•	GitHub = a way to host your content remotely
o	Multiple users can read and/or write versions
o	Everyone uses Git to access it
•	Repositories (Repo) = a place that holds a directory structure (folders) with files
o	Contain all versions and history
o	Typically exist both remotely and locally, although not necessarily in sync
o	Can be created locally with Git and then “pushed” to the remote platform (GitHub)
o	Easier to create a repository in GitHub and then “clone” it locally
